Introduction #

PayFlow is a complete payments infrastructure for the internet. Millions of companies of all sizes—from startups to Fortune 500s—use PayFlow's software and APIs to accept payments, send payouts, and manage their businesses online.

Looking for the API Explorer?

Visit our interactive API playground to test requests against your live data directly in the browser.

Quickstart Guide #

To start using PayFlow, you'll need to install the SDK and initialize it with your API key. We provide libraries for most popular programming languages.

Installing the SDK

Choose your preferred language and run the installation command:

bash 
# Install via npm
npm install @payflow/payflow-js

# Or via yarn
yarn add @payflow/payflow-js

Authentication #

The PayFlow API uses API keys to authenticate requests. You can view and manage your API keys in the PayFlow Dashboard.

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

Keep your Secret Key safe

Never use your secret key in frontend code. Always use the publishable key for client-side operations.

API Key Types

Key Type Prefix Description
Publishable pk_test_ Use in client-side code to tokenize payment info.
Secret sk_test_ Use in server-side code for sensitive operations.
Restricted rk_test_ Granular permissions for specific API resources.

Payment Intents #

A PaymentIntent is an object that represents your intent to collect payment from a customer and tracks the lifecycle of the payment process.

javascript 
const payflow = require('@payflow/payflow-js')('sk_test_51Mz...');

const paymentIntent = await payflow.paymentIntents.create({
  amount: 2000,
  currency: 'usd',
  automatic_payment_methods: {
    enabled: true,
  },
});

console.log(paymentIntent.client_secret);

Handling Webhooks #

Webhooks allow PayFlow to notify your server when an event happens in your account. Webhooks are particularly useful for asynchronous events like customer disputes or successful subscription renewals.

  • Listen for events: Define a webhook endpoint to receive notifications.
  • Verify signatures: Always verify that requests come from PayFlow.
  • Respond promptly: Return a 200 OK status code to acknowledge receipt.
